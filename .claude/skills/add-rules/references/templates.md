## 질문 템플릿

### 규칙 목적 명확화

```markdown
규칙의 목적을 더 명확히 이해하고 싶습니다:

1. 이 규칙이 **해결하려는 문제**는 무엇인가요?
   - 예: 코드 일관성, 버그 방지, 성능 최적화

2. 이 규칙이 **적용되어야 하는 상황**은?
   - 예: 새 파일 생성 시, PR 리뷰 시, 특정 기능 구현 시

3. 규칙을 **따르지 않았을 때**의 결과는?
   - 예: 빌드 실패, 런타임 에러, 유지보수 어려움
```

### 범위 명확화

```markdown
규칙의 적용 범위를 확인하고 싶습니다:

1. **파일 패턴**: 어떤 파일들에 적용되나요?
   - 모든 파일 / 특정 확장자 (예: *.ts) / 특정 디렉토리 (예: src/)

2. **예외 케이스**: 규칙이 적용되지 않아야 하는 경우가 있나요?
   - 예: 테스트 파일, 설정 파일, 레거시 코드

3. **우선순위**: 다른 규칙과 충돌 시 어떻게 처리하나요?
```

### 트리거 명확화

```markdown
규칙의 활성화 조건을 확인하고 싶습니다:

1. **자동 적용**을 원하시나요?
   - 코드 작성할 때마다 자동으로 적용

2. **수동 호출**을 원하시나요?
   - `/{rule-name}` 명령어로 명시적 호출

3. **특정 작업 시**에만 적용하시나요?
   - 예: 코드 리뷰 시, 새 기능 구현 시, 리팩토링 시
```

### 변환 확인

```markdown
기존 규칙 파일을 Skill로 변환하려고 합니다:

1. **소스 파일**: `{source_path}`
2. **변환 후 원본 처리**: 원본을 보존할까요, 삭제할까요?
3. **변환 범위**: 파일 전체를 변환할까요, 일부만 변환할까요?
4. **분리 기준**: 하나의 Skill로 변환할까요, 도메인별로 분리할까요?
```
