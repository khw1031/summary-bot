# MUST 규칙

> 워크플로우 실행 시 반드시 준수해야 하는 필수 규칙입니다.

## 목적

이 디렉토리의 규칙은 워크플로우의 일관성과 품질을 보장하기 위해 **모든 Step에서 필수로 적용**됩니다. 규칙을 위반하면 워크플로우가 실패하거나 예상치 못한 결과가 발생할 수 있습니다.

## 규칙 목록

| 파일                                 | 설명                        |
| ------------------------------------ | --------------------------- |
| [workflow-rule.md](workflow-rule.md) | 워크플로우 실행의 핵심 규칙 |

## 적용 시점

- 워크플로우 시작 시 자동 로드
- 각 Step 전환 시 검증
- 출력 파일 생성 시 형식 검증

## 체크리스트 요약

- [ ] Step 순서 준수 (1→2→3→4→5)
- [ ] 입력 파일 확인 후 진행
- [ ] 출력 템플릿 형식 준수
- [ ] status.yaml 업데이트
- [ ] Git 커밋 체크포인트
